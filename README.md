# RPG To-Do List

RPG To-Do List is a terminal-based (CLI) productivity application that transforms daily task management into Role-Playing Game (RPG) mechanics. It offers an ecosystem where users earn Experience Points (EXP) and in-game currency for completing tasks, which can then be used to purchase items from a virtual marketplace.

## About the Project

This project is a sustainable gamification tool designed to run on the most fundamental computer component—the terminal—without the need for complex interfaces or heavy libraries. Developed in the C programming language, the application is designed with the principles of minimal system resource consumption and maximum portability.

## Key Features

The application features a modular structure comprising the following core systems:

### 1. Quest Management System
* **Add Quest:** Users can create new tasks with a specific name, difficulty level (1-5), reward amount, and detailed descriptions.
* **Auto-Reward Calculation:** Optionally, the system can automatically calculate Gold and EXP rewards based on the assigned difficulty level.
* **Quest Completion:** Completed tasks are listed, rewards are added to the user's balance, and the task is moved to the "Finished Tasks" database.
* **Edit & Delete:** Parameters of existing quests (name, difficulty, reward, etc.) can be modified, or quests can be permanently deleted.

### 2. Market & Economy System
* **Dynamic Marketplace:** Users can purchase items from the market using their earned currency.
* **Market Management:** New products can be added, prices updated, or items removed from the market.
* **Inventory Integration:** Purchased items are automatically transferred to the user's inventory, and the cost is deducted from the balance.

### 3. Inventory Management
* **View Items:** Users can list the items they own, along with their quantities and details.
* **Item Usage:** Items in the inventory can be "used," which decreases their quantity in the inventory.

### 4. Data Persistence (Database)
* All data (quests, inventory, market, finished tasks) is stored in local text files (`.txt`). This ensures data preservation across sessions.

## Future Roadmap

The following features are planned for integration into the system during the development process:

* **XP & Leveling System:** Transforming the current experience point gain into a dynamic Level-Up mechanic.
* **User Management:** Adding multi-user support and a login panel (User Menu).
* **Embedded System Support:** Optimizing the software to run on microcontrollers and development boards like the ESP-32.

## Technical Details & Portability

The primary technical goal of this project is its **dependency-free** architecture.

* **Language & Standard:** Developed in accordance with **C99** standards.
* **Libraries:** Uses only standard C libraries (`stdio.h`, `stdlib.h`, `string.h`, `stdbool.h`). Requires no third-party frameworks or external libraries.
* **Platform:** Can be compiled and run directly on any GNU/Linux and UNIX-based system (including macOS) with a GCC compiler. It utilizes minimal system resources.

## Installation & Usage

To compile and install the project on your system, open a terminal in the project directory and use the following commands.

### Compilation
To compile the project and create the executable `rpg` file:

```bash
make all
```

### Installation (System-Wide)
To install the application system-wide so it can be called from anywhere via the `rpg` command:

```bash
make install
```

### Cleaning
To clean up files generated during compilation:

```bash
make clean
```

# RPG To-Do List (Türkçe)

RPG To-Do List, günlük görev yönetimini bir Rol Yapma Oyunu (RPG) mekaniğine dönüştüren, terminal tabanlı (CLI) bir verimlilik uygulamasıdır. Kullanıcının tamamladığı görevler karşılığında deneyim puanı (EXP) ve oyun içi para birimi kazandığı, bu kazançlarla sanal bir marketten ürünler satın alabildiği bir ekosistem sunar.

## Proje Hakkında

Bu proje, karmaşık arayüzlere veya ağır kütüphanelere ihtiyaç duymadan, en temel bilgisayar bileşeni olan terminal üzerinde çalışan sürdürülebilir bir oyunlaştırma (gamification) aracıdır. C programlama dili ile geliştirilen uygulama, minimum sistem kaynağı tüketimi ve maksimum taşınabilirlik ilkesiyle tasarlanmıştır.

## Temel Özellikler

Uygulama modüler bir yapıya sahiptir ve aşağıdaki temel sistemleri içerir:

### 1. Görev Yönetim Sistemi (Quest System)
* **Görev Ekleme:** Kullanıcılar görev adı, zorluk seviyesi (1-5), ödül miktarı ve detaylı açıklamalar ile yeni görevler oluşturabilir.
* **Otomatik Ödül Hesaplama:** İstenirse, zorluk seviyesine bağlı olarak ödül (Gold) ve EXP miktarı sistem tarafından otomatik hesaplanabilir.
* **Görev Tamamlama:** Tamamlanan görevler listelenir, ödüller kullanıcının bakiyesine eklenir ve görev "Tamamlanmış Görevler" veritabanına taşınır.
* **Düzenleme ve Silme:** Mevcut görevlerin parametreleri (ad, zorluk, ödül vb.) sonradan değiştirilebilir veya görevler tamamen silinebilir.

### 2. Market ve Ekonomi Sistemi
* **Dinamik Market:** Kullanıcılar kazandıkları paralar ile marketten ürün satın alabilirler.
* **Market Yönetimi:** Markete yeni ürün ekleme, fiyat güncelleme veya ürün kaldırma işlemleri yapılabilir.
* **Envanter Entegrasyonu:** Satın alınan ürünler otomatik olarak kullanıcı envanterine aktarılır ve bakiyeden düşülür.

### 3. Envanter Yönetimi
* **Eşya Görüntüleme:** Kullanıcı sahip olduğu eşyaları, miktarlarını ve detaylarını listeleyebilir.
* **Eşya Kullanımı:** Envanterdeki eşyalar "kullanılabilir" ve kullanıldığında envanterden eksilir.

### 4. Veri Kalıcılığı (Database)
* Tüm veriler (görevler, envanter, market, tamamlanan işler) yerel metin dosyalarında (`.txt`) saklanır. Bu sayede uygulama kapatılıp açıldığında veriler korunur.
